# Working Agreement

## Purpose
Maintain a clear, reviewable prototype baseline with explicit capability boundaries.

## Engineering standards
- Describe changes factually and directly.
- State what was verified and what remains unverified.
- Keep product and governance evidence in dedicated folders.
- Avoid introducing production-readiness claims without explicit approval.

## Required change evidence
Every PR description must include:
- What changed
- What was tested
- What is out of scope
- Known risks

Every completion note must include:
- Checks run
- Checks passed
- Checks failed
- Follow-up required

## Scope control
- Keep prototype behavior and production requirements clearly separated.
- Treat external integrations as mocked unless explicitly approved otherwise.
